# Project Zero Product Backlog

## Purpose

This is a product backlog, not a task list.

It describes the future capabilities of Project Zero as an operating system for the Eudlo Road property. Each backlog item should eventually become a module, feature, workflow, dashboard, automation, AI capability, database table, or integration.

## Backlog Prioritisation Rule

Backlog items should be prioritised by:

- Decision risk reduced
- Future work removed
- Financial value created
- Safety improved
- Data foundation unlocked
- Dependency on other modules
- Ease of implementation

If a feature does not improve decisions, reduce work, protect assets, or create measurable value, it should wait.
