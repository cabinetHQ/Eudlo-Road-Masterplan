# Project Principles

## Purpose

These principles guide planning, design, investment, documentation, and construction decisions for Project Eudlo.

## Core Principles

### Design For 20 Years

Decisions should consider long-term property use, business growth, maintenance, expansion, and resale value.

### Avoid Rework

Prefer designs that reduce the chance of digging, rewiring, rebuilding, or repurchasing later.

### Buy Once

When an asset is strategic, prefer a durable and expandable solution over the cheapest short-term option.

### Build Infrastructure Before Capacity

Foundational infrastructure should be planned before adding more load, buildings, machinery, accommodation, or manufacturing capacity.

### Prefer Modular Systems

Use staged, modular systems where possible so Project Eudlo can grow without major redesign.

### Everything Should Be Expandable

Energy, water, buildings, data, communications, machinery, access, and app architecture should all assume future expansion.

### Keep Assumptions Documented

Every major assumption should be visible. Unknowns should be marked as `TBD`, not hidden.

### Challenge Previous Assumptions

Earlier decisions can be reviewed when better data arrives. Do not overwrite history; create a new decision record if the direction changes.

### Separate Facts, Estimates And Decisions

Facts, estimates, assumptions, options, recommendations, and accepted decisions should be distinguishable.

### Prefer Practical Evidence

Use bills, interval data, quotes, site measurements, photos, drawings, supplier advice, accountant advice, and measured performance wherever possible.
